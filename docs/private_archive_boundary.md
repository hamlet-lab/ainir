# Private Archive Boundary

AiNIR has two intentionally separate surfaces.

## Public demo repo

This repository contains:

- small examples;
- a bounded verifier demo;
- a bundled safety registry;
- a bundled evidence ledger;
- a small operation registry;
- negative conformance regression corpus;
- golden trace replay;
- TypeScript skeleton lowering with host enforcement hooks.

## Private archive

The private archive should keep:

- full RC packages;
- historical phase packages;
- extended beta workflows;
- full hardening corpus;
- raw provider-output corpus;
- enterprise policy packs;
- extended evidence/trust ledgers;
- internal evaluation reports.

## Rule

The public repo is for clarity and reputation.
The private archive is for completeness and future product/enterprise work.
