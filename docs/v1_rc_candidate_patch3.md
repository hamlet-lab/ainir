# AiNIR v1.0 RC Candidate Patch 3 - Scope Boundaries and Extensibility Roadmap

Patch 3 documents the intentional boundaries of the RC candidate.

It does not add production integration and does not expand the public demo into a general-purpose enterprise verifier.

## What changed

- Bounded workflow registry is documented.
- Evidence provider interface roadmap is documented.
- Canonical effect taxonomy roadmap is documented.
- TrustReceipt registry evolution modes are documented.
- Executable draft metadata semantics are clarified.
- VerifiedIntentPacket export scope is clarified.
- Unknown workflow findings now suggest registering a workflow profile instead of silently implying arbitrary workflow support.

## Why this patch exists

The RC candidate is strong inside its bounded registry. It should not overclaim outside that boundary.

This patch makes the boundary explicit so the public repo reads as a serious RC candidate rather than an overbroad production promise.
