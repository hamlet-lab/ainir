# Contributing

The most important rule:

> Please do not weaken gates simply to make a draft pass.

AiNIR is useful only if unsafe semantics remain visible and blocked.

Good contributions are:

- clearer unsafe draft examples,
- safer policy predicates,
- smaller reproductions of model-generated hazards,
- better documentation,
- tests that ensure unsafe semantics remain blocked.

Avoid contributions that silently convert an unsafe draft into a passing draft without an explicit finding, downgrade, repair, or review step.
